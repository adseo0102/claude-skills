# /competitor — 경쟁사 분석 Step 워크플로우 파이프라인

사용자가 $ARGUMENTS 로 분석 요청을 전달합니다.

### 입력 모드 (2가지)

**모드 A — 브랜드 직접 지정:**
```
/competitor 농심(신라면) vs 오뚜기(진라면) vs 삼양식품(불닭볶음면)
```
→ 지정된 브랜드끼리 비교 분석

**모드 B — 브랜드 1개만 입력 (자동 경쟁사 탐색):**
```
/competitor 신라면
```
→ 해당 브랜드의 카테고리를 WebSearch + Gemini로 파악
→ 메인 SKU/서비스 기준 카테고리 내 TOP 3 경쟁 브랜드를 자동 선정
→ 선정 결과를 사용자에게 보여주고 확인 후 분석 시작

아래 4-Step 파이프라인을 **반드시 순서대로** 실행하세요. 단계를 건너뛰지 마세요.

---

## 사전 단계: 요구사항 파싱

### 입력 모드 판별

1. `vs`, `VS`, `대비`, `비교` 키워드 포함 → **모드 A** (브랜드 직접 지정)
2. 브랜드/제품 1개만 입력 → **모드 B** (자동 경쟁사 탐색)

### 모드 B: 자동 경쟁사 탐색

브랜드 1개만 입력된 경우:
1. WebSearch: `"{브랜드명}" 카테고리 시장 점유율 TOP 경쟁사`
2. Gemini 교차검증: `node ~/gemini-tools/gemini-search.mjs "{브랜드명} 카테고리 주요 경쟁사 TOP 3"`
3. 메인 SKU/서비스 기준으로 동일 카테고리 TOP 3 브랜드 선정
4. 선정 결과를 사용자에게 보여주고 확인:

```
## 🔍 경쟁사 자동 탐색 결과

입력 브랜드: {브랜드명}
감지된 카테고리: {카테고리명}
메인 SKU: {메인 제품/서비스}

### 선정된 비교 대상 TOP 3
| 순위 | 브랜드 | 메인 SKU | 선정 근거 |
|------|--------|----------|----------|
| 1 | {브랜드A} | {제품A} | 카테고리 점유율 1위 |
| 2 | {브랜드B} | {제품B} | 직접 경쟁 (동일 타겟) |
| 3 | {브랜드C} | {제품C} | 급성장 (YoY +XX%) |

이 3개 브랜드로 비교 분석을 진행할까요?
수정이 필요하면 알려주세요.
```

승인 시 모드 A와 동일하게 Step 1로 진행.

### 파싱 결과 출력

```
## 📋 분석 요구사항 파싱

| 항목 | 추출 결과 |
|------|----------|
| 입력 모드 | A (직접 지정) / B (자동 탐색) |
| 분석 대상 | (브랜드 리스트) |
| 비교 기준 | (사용자 지정 또는 기본: 시장점유율, 마케팅, SNS, 해외) |
| 분석 깊이 | 🟢 기본 / 🟡 심화 / 🔴 딥리서치 |
| 출력 형태 | (대시보드 + 경영진 요약) |
```

### 분석 깊이 자동 판단 기준
- **기본 (🟢)**: 경쟁사 1~2개, 비교 기준 3개 이하 → Step 1에서 모듈당 검색 2~3회
- **심화 (🟡)**: 경쟁사 3개 이상 또는 비교 기준 4개 이상 → 모듈당 검색 3~4회
- **딥리서치 (🔴)**: "딥리서치", "심층", "정밀" 키워드 포함 → 모듈당 검색 4~5회 + Gemini 딥리서치

### 자사 브랜드 감지 (선택적)
다음 키워드가 포함되면 자사로 인식하여 SWOT/액션 플랜에서 "자사 관점" 분석 추가:
- CCFM, 씨씨에프엠, 메라블, melable, 루비알엔, RubyRN
- "우리", "자사", "당사"

자사 미감지 시: 모든 브랜드를 동등하게 비교하는 **객관 비교 모드**로 진행. 액션 플랜 탭은 "각 브랜드별 전략 시사점"으로 대체.

### 조기 프레임 바인딩

사용자 프롬프트에서 프레임 키워드를 감지:
- "SWOT" → 프레임 확정, 모듈 A~E 전체
- "4P" → 프레임 확정, 모듈 A+C+D만 실행
- "퍼널", "AARRR" → 프레임 확정, 모듈 C+D+E만 실행
- "포지셔닝" → 프레임 확정, 모듈 A+B+C만 실행
- "미디어 믹스" → 프레임 확정, 모듈 D+E만 실행
- 키워드 없음 → 기존 플로우 (Step 1 전체 → Step 2 추천)

프레임 확정 시: Step 2를 건너뛰고 바로 Step 2.5 → Step 3로 진행.
파싱 결과에 "감지된 프레임: {프레임명} → 모듈 {X,Y,Z}만 실행" 표시.

파싱 결과를 사용자에게 보여주고, 맞는지 확인합니다. 수정 없이 "좋아", "OK" 등이면 Step 1로 진행합니다.

## Step 1: 딥 리서치

5개 리서치 모듈을 순차 실행하여 각 경쟁사의 데이터를 수집합니다.

### 모듈 A: 전략적 포지션

⚠️ 안티패턴 금지: "OO사는 20XX년에 설립된..." 식의 연혁 나열 금지.

**검색 키워드 예시:**
- `"{경쟁사명}" 핵심 경쟁력 차별화 USP`
- `"{경쟁사명}" 비즈니스 모델 수익 구조`
- `"{경쟁사명}" 최근 전략 변화 신사업`

**수집 항목:**
| 항목 | 내용 | 안티패턴 체크 |
|------|------|-------------|
| 핵심 경쟁 무기 | 이 회사가 시장에서 이기는 이유 1가지 | "다양한 서비스" ← 금지 |
| 비즈니스 모델 | 수익 구조 (수수료/구독/광고/커머스) | "사업 영역이 넓다" ← 금지 |
| 최근 전략 방향 | 최근 6개월 내 주요 전략 변화 | 설립 연도 나열 ← 금지 |
| 규모 지표 | MAU, GMV, 직원 수 중 가용한 것 | 비전 슬로건 나열 ← 금지 |

### 모듈 B: 재무/실적

**검색 키워드 예시:**
- `"{경쟁사명}" 매출 영업이익 2024 2025`
- `"{경쟁사명}" 투자 유치 시리즈 벨류에이션`
- `"{경쟁사명}" 성장률 시장 점유율`

**수집 항목:**
| 항목 | 내용 |
|------|------|
| 매출/실적 | 최근 2~3년 매출, 영업이익 |
| 성장률 | YoY 성장률 |
| 투자/펀딩 | 투자 라운드, 금액, 투자사 |
| 시장 점유율 | 카테고리 내 점유율 |

### 모듈 C: 고객/타겟

**검색 키워드 예시:**
- `"{경쟁사명}" 타겟 고객 페르소나`
- `"{경쟁사명}" 리뷰 평점 사용자 반응`
- `"{경쟁사명}" 고객 사례 성공 스토리`

**수집 항목:**
| 항목 | 내용 |
|------|------|
| 타겟 고객층 | 연령, 성별, 직업군 |
| 핵심 페르소나 | 주요 사용자 유형 |
| 리뷰/평판 | 주요 플랫폼 평점, 핵심 키워드 |
| 고객 사례 | 대표 성공 사례 |

### 모듈 D: 마케팅 전략

**검색 키워드 예시:**
- `"{경쟁사명}" 마케팅 전략 광고 채널`
- `"{경쟁사명}" 크리에이티브 캠페인 콘텐츠`
- `"{경쟁사명}" 인플루언서 협찬 마케팅`
- `"{경쟁사명}" 미디어 예산 광고비`

**수집 항목:**
| 항목 | 내용 |
|------|------|
| 미디어 믹스 | 주요 광고 채널, 비중 |
| 크리에이티브 | 광고 톤앤매너, 핵심 메시지 |
| 콘텐츠 전략 | 블로그, SNS, 뉴스레터 |
| 인플루언서 | 협업 패턴, 규모 |
| 프로모션 | 할인, 이벤트, 멤버십 |

### 모듈 E: 디지털 프레전스

**검색 키워드 예시:**
- `"{경쟁사명}" 앱 다운로드 수 순위 앱스토어`
- `"{경쟁사명}" SEO 검색 트래픽 키워드`
- `"{경쟁사명}" 인스타그램 팔로워 유튜브 구독자`

**수집 항목:**
| 항목 | 내용 |
|------|------|
| 앱 순위 | 앱스토어/구글플레이 순위, 다운로드 수 |
| SEO | 주요 키워드, 검색 볼륨, 도메인 권한 |
| SNS 현황 | 채널별 팔로워, 게시 빈도, 인게이지먼트 |
| 웹 트래픽 | 월간 방문자, 트래픽 소스 |

### 경쟁사 유형 분류 (사전 단계 파싱 시 결정)

| 유형 | 예시 | 중점 모듈 (검색 +1회) | 경량 모듈 (검색 -1회) |
|------|------|---------------------|---------------------|
| D2C 브랜드 | 자체 제조/판매 | C 고객, D 마케팅, E 디지털 | B 재무 (비상장 다수) |
| 플랫폼/유통사 | 올리브영, 쿠팡 | A 포지션, B 재무 | - |
| 에이전시/서비스 | 광고대행사 | A 포지션, C 고객 | E 디지털 (앱 없음) |
| 스타트업 | 초기 단계 | B 재무(투자), E 디지털 | - |

타 경쟁사와 동일한 문장이 2개 이상 반복되면 안티패턴으로 간주.

### 리서치 실행 규칙

1. **경쟁사별로 모듈 A→E 순차 실행** (각 모듈당 분석 깊이에 따라 2~5회 검색)
2. **WebSearch로 검색** — 한국어 + 영어 키워드 병행
3. **Gemini 교차검증** — 각 모듈 완료 후 1회씩 Gemini로 핵심 수치 검증:
   ```bash
   node ~/gemini-tools/gemini-search.mjs "{경쟁사명} {모듈 주제} 핵심 데이터"
   ```
4. **검증 표기**: ✅ WebSearch+Gemini 일치 | 🔍 WebSearch만 | 🤖 Gemini만 | ⚠️ 불일치
5. **딥리서치 모드(🔴)** 선택 시, 전체 리서치 완료 후 Gemini 딥리서치 추가:
   ```bash
   node ~/gemini-tools/gemini-deep-research.mjs "{자사} vs {경쟁사} 비교 분석"
   ```

### 시계열 비교 (자동)

- Step 1 완료 시 리서치 결과를 JSON으로 저장:

- Step 1 시작 시 이전 분석 파일 존재 여부 확인

  - 🆕 신규: 새 캠페인, 신제품, 채널 진입
- 대시보드에 "vs 이전 분석" 토글 추가 (이전 데이터 있을 때만 표시)

## Step 2: 분석 프레임 선택

리서치 데이터를 기반으로 적합한 분석 프레임을 추천합니다.

### 사용 가능한 6종 프레임

#### 1. SWOT 분석
- **적합**: 자사 vs 경쟁사 전략 수립 시
- **필요 데이터**: 모듈 A~E 전체
- **결과물**: 경쟁사별 SWOT 매트릭스 + 자사 전략 시사점

#### 2. 4P 분석 (Product, Price, Place, Promotion)
- **적합**: 마케팅 믹스 비교 시
- **필요 데이터**: 모듈 A(서비스), D(마케팅), C(고객)
- **결과물**: 경쟁사별 4P 비교표 + 갭 분석

#### 3. 퍼널 분석 (AARRR)
- **적합**: 고객 여정/전환 비교 시
- **필요 데이터**: 모듈 C(고객), D(마케팅), E(디지털)
- **결과물**: 퍼널 단계별 경쟁사 전략 비교

#### 4. 포지셔닝 맵
- **적합**: 시장 내 위치 시각화 시
- **필요 데이터**: 모듈 A(서비스), B(재무), C(고객)
- **결과물**: 2축 포지셔닝 맵 + 경쟁사 좌표 분석
- **축 후보**: 가격↔프리미엄, 대중↔니치, 온라인↔오프라인, 브랜드↔퍼포먼스

#### 5. 미디어 믹스 비교
- **적합**: 광고/마케팅 채널 전략 비교 시
- **필요 데이터**: 모듈 D(마케팅), E(디지털)
- **결과물**: 채널별 비중 비교 차트 + 투자 대비 효과 분석

#### 6. 커스텀 프레임
- **적합**: 특정 비교 축이나 독자적 분석 관점이 있는 경우
- **사용법**: 비교 축 2개 또는 분석 관점을 자연어로 기술
- **처리**: 사용자 기술 기반으로 비교 항목과 시각화 유형 자동 설계

### 프레임 자동 선택 + 전체 실행

**사용자가 프레임을 지정한 경우**: 지정된 프레임 전부를 한번에 실행.
**사용자가 프레임을 지정하지 않은 경우**: 데이터 충분도를 기반으로 적합한 프레임을 자동 선택하여 전부 실행.

#### 자동 선택 로직

데이터 충분도에 따라 실행 가능한 프레임을 자동 판별:

| 조건 | 자동 선택 프레임 |
|------|---------------|
| 모듈 A~E 전체 충분 | SWOT + 미디어 믹스 + 포지셔닝 맵 (3개) |
| 모듈 D, E 부족 | SWOT + 4P (2개) |
| 자사 브랜드 감지 | SWOT 필수 포함 |
| 사용자가 명시적 지정 | 지정된 프레임 전부 |

#### 핵심 변경: 체크포인트 ② 생략

**프레임 선택을 위한 별도 승인 체크포인트는 없습니다.**
조기 프레임 바인딩(사전 단계)에서 확정되었거나, 자동 선택 로직으로 결정됩니다.

체크포인트 ①(리서치 검토)에서 승인받을 때, 실행할 프레임 목록도 함께 표시:

```
## ✅ Step 1 리서치 완료

### 데이터 충분도
(기존과 동일)

### 실행할 분석 프레임
✅ SWOT 분석 — 모듈 A~E 데이터 충분
✅ 미디어 믹스 — 모듈 D, E 데이터 충분
✅ 포지셔닝 맵 — 모듈 A, B, C 데이터 충분

이대로 3개 프레임 전체 실행 + 대시보드 제작으로 넘어갈까요?
```

사용자가 "OK" 하면 Step 2.5(변환) → Step 3(대시보드)로 바로 진행.
프레임을 빼거나 추가하고 싶으면 이 시점에 요청.

## Step 2.5: 프레임 → 액션 변환 규칙

분석 프레임 결과를 액션 후보로 변환한다. 변환 없이 Step 3에 진입하지 마라.

### SWOT → 액션 매핑
| SWOT 셀 | 변환 규칙 | 액션 유형 |
|----------|-----------|-----------|
| S+O (강점×기회) | 강점을 레버리지하여 기회 선점 | 공격적 투자/확장 |
| W+O (약점×기회) | 약점 보완 후 기회 포착 | 역량 확보/파트너십 |
| S+T (강점×위협) | 강점으로 위협 방어 | 방어적 차별화 |
| W+T (약점×위협) | 리스크 최소화/회피 | 철수/피벗/헷지 |

### 4P → 액션 매핑
| 갭 유형 | 변환 규칙 |
|----------|-----------|
| Price 갭 > 20% | 가격 전략 재검토 액션 생성 |
| Place 커버리지 부족 | 채널 확장 액션 생성 |
| Promotion 강도 열위 | 미디어 예산 재배분 액션 생성 |

### 검증
액션 플랜의 모든 항목은 반드시 소스 프레임의 특정 셀/항목을 괄호로 참조해야 함.
예: "인스타그램 릴스 주 3회 → 월 50만 → 4주 내 → 팔로워 +15% (SWOT: S2×O1)"

## Step 3: 분석 대시보드 제작

### /viz 연동 규칙

competitor 스킬은 /viz 스킬을 직접 호출하지 않는다.
/viz의 HTML 스켈레톤 구조, CCFM 브랜드 변수, Chart.js 패턴만 차용한다.
탭 설계, 필터링, 데이터 바인딩, 액션 플랜 렌더링은 competitor가 직접 담당.

/viz 스킬의 스켈레톤 템플릿과 CCFM 브랜드 규칙을 적용하여 대시보드를 제작합니다.

### 대시보드 구조

```
[헤더] 제목 + 분석 일자 + 비교 대상

[탭 네비게이션]
  ├── 탭1: 요약 (Executive Summary)
  ├── 탭2~N: 선택된 프레임별 분석
  └── 마지막 탭: 액션 플랜

[필터 바] 경쟁사 선택 필터 (전체 / 개별 토글)
```

### 필수 섹션

#### 1. Executive Summary 탭
- KPI 카드 그리드 (비즈니스 지표 중심, 메타 정보 KPI 금지):
  - 자사 vs 경쟁 평균 시장점유율 갭 (모듈 B)
  - 자사 vs 경쟁 평균 디지털 도달률 (모듈 E)
  - 자사 vs 경쟁 평균 SNS 인게이지먼트 비율 (모듈 E)
  - 가장 큰 기회 영역 1가지 + 위협 영역 1가지
  - 데이터 미확보 시 "N/A" 표기, 더미 수치 금지
- 한눈에 보는 비교 매트릭스 (스코어카드)
- 핵심 인사이트 3가지 (하이라이트 카드)

#### 2. 분석 프레임 탭 (선택된 프레임에 따라)

**SWOT 탭 구성:**
- 경쟁사별 SWOT 4분면 카드
- 각 SWOT 셀에 근거 수치/출처 1개 이상 필수 (예: "S1: SNS 팔로워 120만 — 인스타그램 2026.03")
- 강점/약점 점수 레이더 차트 (Chart.js radar)
- 자사 vs 경쟁사 비교 하이라이트

**4P 탭 구성:**
- Product/Price/Place/Promotion 비교 매트릭스
- 가격대 비교 바 차트
- 채널 분포 도넛 차트

**퍼널 탭 구성:**
- AARRR 단계별 경쟁사 전략 비교 카드
- 퍼널 시각화 (상단 넓음 → 하단 좁음)

**포지셔닝 맵 탭 구성:**
- 2축 산점도 (Chart.js scatter)
- 축 설명 + 인사이트 카드
- 경쟁사 좌표 설명

**미디어 믹스 탭 구성:**
- 채널별 비중 스택 바 차트
- 채널별 세부 카드 (SNS, 검색광고, 디스플레이 등)
- 추정 예산 비교 (가능한 경우)

#### 3. 액션 플랜 탭 ("그래서 우리는?")

각 액션은 반드시 아래 필드를 모두 포함해야 한다:

| 필드 | 필수 | 설명 |
|------|------|------|
| 채널 | ✅ | 실행할 구체적 채널 (예: 메타 리타겟팅, 네이버 SA) |
| 예산 범위 | ✅ | 월 단위 예상 예산 (예: 500~800만원/월) |
| 타임라인 | ✅ | 시작~완료 기간 (예: 2026 Q2) |
| KPI | ✅ | 측정 지표 + 목표치 (예: ROAS 300% 이상) |
| 근거 프레임 | ✅ | Step 2.5 변환 출처 (예: SWOT S2×O1) |

- **즉시 액션 (1~2주)**: 기존 리소스로 바로 실행 가능한 3가지
- **단기 액션 (1~3개월)**: 예산/리소스 확보 후 실행
- **중장기 로드맵 (3~12개월)**: 분기별 전략 방향

**⛔ 액션 안티패턴 (아래 수준의 추상 표현 금지):**
- "디지털 마케팅 강화", "브랜드 인지도 제고", "고객 경험 개선", "SNS 활성화"
- 반드시 채널·예산·기한·KPI가 모두 채워진 상태에서만 액션으로 인정

### 기술 요구사항

1. **viz.md 스켈레톤 사용**: 반드시 /viz 스킬의 HTML 스켈레톤에서 시작
2. **Chart.js 활용**: `Chart.defaults.animation = false;` 필수
3. **탭 네비게이션**: 클릭으로 프레임 전환, URL 해시로 딥링크 지원
4. **필터링**: 경쟁사 토글 버튼으로 특정 경쟁사 표시/숨기기
5. **반응형**: 모바일에서도 사용 가능
6. **CCFM 브랜드**: `#e5007e` accent, Pretendard 폰트
7. **인터랙티브 요소**:
   - 차트 호버 툴팁
   - 탭 전환 애니메이션
   - 필터 토글
   - 카드 호버 피드백
   - 스크롤 리빌 (IntersectionObserver)

### 파일 저장
- **파일명**: `~/Downloads/{자사}-vs-{경쟁사}-경쟁분석-대시보드.html`
- **예시**: `~/Downloads/메라블-vs-올리브영-경쟁분석-대시보드.html`

### 경영진용 요약 슬라이드 (별도 HTML, 대시보드 없이 독립 활용 가능)

**슬라이드 1: 분석 개요**
- 분석 배경/목적, 비교 대상, 분석 일자, 사용된 프레임

**슬라이드 2: 시장 포지션 한눈에 보기**
- 핵심 비즈니스 KPI 비교 테이블 (매출/트래픽/SNS 등 정량 지표)
- 자사 위치 하이라이트

**슬라이드 3: 핵심 발견 3가지**
- 각 발견에 근거 데이터 1줄 + So What(시사점) 1줄 포함

**슬라이드 4: 액션 플랜 요약**
- 즉시/단기/중장기 각 1개씩 (채널+예산+KPI 포함)

**슬라이드 5: 리스크 & 모니터링 포인트**
- 주시해야 할 경쟁사 움직임 2~3가지
- 다음 분석 권장 시점

파일명: ~/Downloads/{자사}-vs-{경쟁사}-경영진요약.html

## 에러 처리

- **검색 실패**: 다른 키워드로 재시도 (최대 2회), 그래도 실패 시 해당 항목 "데이터 미확보" 표시
- **Gemini 실패**: WebSearch 결과만으로 진행 (에러 무시)
- **데이터 부족**: 체크포인트 ①에서 부족 영역 명시, 사용자와 대체 방안 논의
- **비상장사 재무**: "비상장사로 공시 정보 제한" 표기 + 언론 보도 추정치 활용

## 콘텐츠 규칙

- 모든 수치에 출처 명시
- 추정치는 "~추정", "약" 등 표기
- 비교는 객관적 데이터 기반, 주관적 평가 최소화
- 액션 플랜은 구체적이고 실행 가능한 수준으로 작성
- 자사 브랜드 관점에서의 시사점을 반드시 포함
