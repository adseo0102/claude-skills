# REVIEW_SPEC — /cardnews 카드뉴스 제작 파이프라인

## 프로젝트 목적
CCFM 마케터/브랜드 담당자 대상, 퍼포먼스 마케팅·광고 운영 주제의 카드뉴스를 리서치→기획→제작→배포까지 자동화하는 파이프라인.

## 핵심 품질 기준 (MUST)
이 기준을 충족하지 않으면 🔴 critical:

1. **리서치 팩트 검증**: 최소 **3개 출처** 수집, 핵심 수치는 **2회 이상 교차 검증**(WebSearch + Gemini). 단일 출처 수치를 그대로 사용하면 안 됨
2. **슬라이드 수**: 커버+CTA 포함 **8~12장**. 7장 이하는 정보 부족, 13장 이상은 이탈률 증가
3. **슬라이드 텍스트 밀도**: 본문 **40자/줄, 최대 3줄 = 120자 이내**. 한 슬라이드에 2개 이상의 독립된 메시지 금지
4. **디자인 시스템 준수**: 1080x1080px 캔버스, **폰트: Pretendard (대체: Noto Sans KR)**, CCFM 컬러 팔레트(#e5007e 메인), padding 60px, border-radius 16px — 하나라도 이탈하면 브랜드 일관성 훼손
5. **폰트 크기 계층**: 제목 **48px 이상**, 본문 **28px 이상**, 캡션 **20px 이상**. 이보다 작으면 모바일에서 가독성 훼손
6. **Hook → 전개 → CTA 구조**: 첫 슬라이드가 훅이 아니거나, 마지막 슬라이드에 CTA가 없으면 콘텐츠 완결성 부재
7. **텍스트 잘림/깨짐 없음**: PNG 변환 후 텍스트가 잘리거나, 레이아웃이 깨지거나, 폰트가 시스템 기본으로 대체되면 안 됨

## 권장 품질 기준 (SHOULD)
충족하면 품질 향상, 미충족 시 🟡 warning:

1. **배경 교차**: 동일 배경 **연속 2장까지** 허용. **3장 연속 시 🔴**. 다크↔라이트↔포인트 교차로 시각적 리듬 유지
2. **데이터↔텍스트 교차 배치**: 숫자/차트 슬라이드와 서술 슬라이드가 번갈아 나와야 읽기 피로 감소
3. **구체적 수치/사례 포함**: "많이 증가했다" 대신 "+340% YoY" 같은 정량 표현 우선
4. **캐러셀 뷰어 UX**: Vercel 배포 시 스와이프+키보드+도트 네비게이션 모두 작동, 모바일 대응

## 폰트 규칙
- **Primary**: Pretendard (CDN: `https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable.min.css`)
- **Fallback**: Noto Sans KR (CDN: `https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;500;700;800&display=swap`)
- Pretendard 로드 실패 시에만 Noto Sans KR 적용. 그 외 시스템 폰트(-apple-system 등)는 최후 대체

## 안티패턴
이렇게 하면 안 됨:

1. **리서치 없이 기획 진입** — 팩트 없는 카드뉴스는 블로그 썸네일 수준. 반드시 Step 1 완료 후 진행
2. **승인 없이 제작** — 기획안 승인(Step 3) 없이 HTML 제작으로 넘어가면 재작업 비용 폭증
3. **기획안에 없는 내용 임의 추가** — 제작 단계에서 "이것도 넣으면 좋겠다"는 금지. 기획안이 진실의 원천
4. **이모지 남발** — 태그/라벨에 이모지 쓰지 않음. 시스템 이모지는 TIP 박스(💡)에만 제한적 사용
5. **외부 이미지 의존** — CDN 이미지 링크가 깨지면 카드뉴스 전체가 무용지물. CSS + 인라인 SVG로 해결
6. **Pretendard/Noto Sans KR 외 폰트 사용** — 브랜드 일관성을 위해 다른 폰트 금지
