# /strategy — 실행 전략 설계 파이프라인

사용자가 $ARGUMENTS 로 전략 설계 요청을 전달합니다.

### 입력 모드 (3가지)

**모드 A — /competitor 연계 (가장 강력):**
```
/strategy [브랜드명] — /competitor 분석 결과 기반
```
→ ~/Downloads/ 에서 해당 브랜드의 경쟁분석 대시보드 HTML을 탐색
→ 분석 데이터를 자동 추출하여 전략 설계의 입력으로 사용
→ ~/.claude/data/competitor/ 디렉토리에서 JSON 데이터도 확인

**모드 B — 브랜드/제품 직접 입력:**
```
/strategy 메라블 루비알엔 — 국내 시장 확대 전략
```
→ WebSearch + Gemini로 브랜드/제품/시장 데이터를 자체 수집 후 전략 설계

**모드 C — 주제/과제 기반:**
```
/strategy 홈뷰티 디바이스 시장 진입 전략
```
→ 시장/카테고리 수준의 리서치 후 전략 프레임워크 적용

아래 6-Step 파이프라인을 **반드시 순서대로** 실행하세요. 단계를 건너뛰지 마세요.

---

## 사전 단계: 요구사항 파싱

### 입력 모드 판별

1. `/competitor` 결과 참조 키워드 포함 ("경쟁분석 기반", "competitor 결과로" 등) → **모드 A**
2. 구체적 브랜드/제품명 포함 → **모드 B**
3. 시장/카테고리/과제 수준 → **모드 C**

### 모드 A: /competitor 데이터 로드

1. `~/Downloads/` 에서 `*경쟁분석*대시보드*.html` 패턴 탐색
2. `~/.claude/data/competitor/` 에서 최신 JSON 데이터 로드
3. 경쟁사 분석 결과 요약을 전략 설계 입력으로 바인딩:
   - 경쟁사별 SWOT/포지셔닝 데이터 → Step 2 매슬로 분석에 활용
   - 경쟁사 마케팅 전략 데이터 → Step 4 드림100에 활용
   - 액션 플랜 데이터 → Step 5 통합 전략에 반영

### 자사 브랜드 감지

다음 키워드가 포함되면 자사로 인식:
- CCFM, 씨씨에프엠, 메라블, melable, 루비알엔, RubyRN, 유쎄라블, Ucerable
- "우리", "자사", "당사"

자사 감지 시: 준석님의 기획 8대 원칙(memory 참조)과 연계하여 전략 일관성 검증 추가.

### 파싱 결과 출력

```
## 📋 전략 설계 요구사항 파싱

| 항목 | 추출 결과 |
|------|----------|
| 입력 모드 | A (competitor 연계) / B (브랜드 직접) / C (과제 기반) |
| 전략 대상 | (브랜드/제품/시장) |
| /competitor 데이터 | ✅ 로드됨 / ❌ 없음 |
| 자사 브랜드 | ✅ 감지 / ❌ 미감지 |
| 전략 목적 | (시장 진입 / 점유율 확대 / 신제품 론칭 / 리브랜딩 등) |
| 분석 깊이 | 🟢 기본 / 🟡 심화 / 🔴 딥리서치 |
```

파싱 결과를 사용자에게 보여주고 확인합니다. "좋아", "OK" 등이면 Step 1로 진행합니다.

## Step 2: 매슬로 5대 욕구 분석 — "어떤 욕망을 건드릴 것인가"

### 프레임워크 개요

소비자의 구매 동기를 매슬로 5단계로 분석하여, 브랜드/제품이 건드려야 할 핵심 욕구를 식별한다.

### 5대 욕구 × 마케팅 전략 매핑

| 단계 | 욕구 | 마케팅 전략 방향 | 메시지 패턴 |
|------|------|-----------------|-----------|
| 1. 생리적 욕구 | 기본 생존, 건강, 편의 | 제품의 기본 기능/효능 소구 | "이것 없이는 OO가 불가능합니다" |
| 2. 안전 욕구 | 안전, 보호, 리스크 회피 | 신뢰성, 인증, 보장, 성분 안전성 | "검증된 OO로 안심하세요" |
| 3. 소속/사랑 욕구 | 소속감, 커뮤니티, 관계 | 커뮤니티, UGC, 인플루언서, 사회적 증거 | "OO만 명이 이미 사용 중" |
| 4. 존경 욕구 | 인정, 지위, 전문성, 성취 | 프리미엄 포지셔닝, 한정판, 전문가 인증 | "당신의 안목이 증명됩니다" |
| 5. 자아실현 욕구 | 자기표현, 성장, 의미 | 브랜드 철학, 가치 소비, 맞춤화 | "당신만의 OO를 완성하세요" |

### 분석 실행

각 욕구 단계에 대해:
1. **현재 상태**: 브랜드/제품이 현재 건드리고 있는 욕구 레벨 식별
2. **경쟁사 비교**: 경쟁사는 어떤 욕구에 집중하는지 (/competitor 데이터 활용)
3. **갭 분석**: 아직 건드리지 못한 욕구 레벨 식별
4. **우선순위**: 매출 임팩트가 가장 큰 욕구 레벨 2~3개 선정
5. **메시지 방향**: 각 욕구에 맞는 메시지 프레임 도출

### 안티패턴

- ⛔ 모든 5단계를 균등하게 다루는 것 → 핵심 2~3단계에 집중
- ⛔ 단순 분류로 끝나는 것 → "그래서 어떤 메시지를?"까지 도달
- ⛔ 제품 카테고리와 무관한 욕구 강제 매핑 → 카테고리별 자연스러운 욕구 구조 존중

### 출력 형식

```
## 🧠 매슬로 5대 욕구 분석: {브랜드명}

### 현재 욕구 포지션
[5단계 시각화 — 현재 강점 영역 하이라이트]

### 핵심 욕구 타겟 (우선순위)
| 순위 | 욕구 단계 | 전략 방향 | 기대 효과 | 실행 메시지 예시 |
|------|----------|----------|----------|---------------|

### 경쟁사 욕구 포지션 비교
| 브랜드 | 주력 욕구 | 메시지 특징 | 갭/기회 |
|--------|----------|-----------|--------|

### 💡 인사이트
- (핵심 발견 3가지)
```

## Step 3: 피터 드러커 "고객 창조" 체크 — "고객을 만들어내고 있는가"

### 프레임워크 개요

드러커의 핵심 질문: "기업의 목적은 고객 창조이며, 마케팅과 혁신만이 결과를 만든다."
이 관점에서 현재 전략이 진정한 고객 창조를 하고 있는지 진단한다.

### 5대 진단 질문

| # | 드러커 질문 | 진단 항목 | 평가 기준 |
|---|-----------|----------|----------|
| 1 | "고객은 누구인가?" | 타겟 고객 정의의 선명도 | 인구통계만 ❌ → 행동/맥락/욕구 기반 ✅ |
| 2 | "고객이 가치로 여기는 것은?" | 고객 인식 가치 vs 우리가 전달하는 가치 | 기능 나열 ❌ → 고객 언어로 번역된 가치 ✅ |
| 3 | "마케팅이 판매를 불필요하게 만드는가?" | 제품-시장 적합도 수준 | 설득이 필요 ❌ → 알아서 팔림 ✅ |
| 4 | "혁신이 새 고객을 만드는가?" | 기존 고객 유지만 vs 새 시장 창출 | 리텐션만 ❌ → 카테고리 확장 ✅ |
| 5 | "결과를 만드는 활동에 집중하는가?" | 마케팅+혁신 vs 나머지 비용의 비율 | 관리 비용 과다 ❌ → 고객 접점 투자 ✅ |

### 분석 실행

각 질문에 대해:
1. **현재 상태 진단** (🟢 우수 / 🟡 보통 / 🔴 미흡)
2. **근거 데이터** (리서치/competitor 데이터에서 추출)
3. **개선 방향** (구체적 액션 1~2개)

### 고객 창조 스코어카드

```
## 📊 드러커 고객 창조 스코어카드: {브랜드명}

| 진단 항목 | 현재 수준 | 근거 | 개선 방향 |
|----------|---------|------|----------|
| 고객 정의 선명도 | 🟢/🟡/🔴 | ... | ... |
| 고객 가치 번역 | 🟢/🟡/🔴 | ... | ... |
| PMF 수준 | 🟢/🟡/🔴 | ... | ... |
| 혁신→고객 창출 | 🟢/🟡/🔴 | ... | ... |
| 결과 활동 집중도 | 🟢/🟡/🔴 | ... | ... |

### 종합 평가
총점: X / 5 (🟢 4+: 고객 창조 엔진 가동 중 / 🟡 2~3: 부분 작동 / 🔴 0~1: 근본 재설계 필요)

### 💡 핵심 개선 포인트
1. (가장 시급한 개선 1개)
2. (단기 레버리지 1개)
3. (중장기 혁신 방향 1개)
```

## Step 4: 러셀 브런슨 3부작 프레임워크 — "어떻게 팔 것인가"

### 4-A. 가치 사다리 (Value Ladder) 설계

**출처: DotCom Secrets**

고객을 무료/저가에서 고가까지 자연스럽게 이동시키는 오퍼 체계를 설계한다.

| 단계 | 가격대 | 역할 | 예시 형태 |
|------|--------|------|----------|
| 미끼 (Bait) | 무료~저가 | 리드 획득, 첫 접점 | 무료 샘플, 리드 마그넷, 체험단, 무료 콘텐츠 |
| 프론트엔드 (Frontend) | 저~중가 | 첫 구매 전환, 신뢰 구축 | 히어로 SKU, 트라이얼 세트, 입문 제품 |
| 미들 (Middle) | 중가 | 반복 구매, 관계 심화 | 정기 구독, 풀라인 세트, 업그레이드 |
| 백엔드 (Backend) | 고가 | LTV 극대화, 프리미엄 | 프리미엄 라인, VIP 멤버십, 디바이스 |
| 꼭대기 (Pinnacle) | 최고가 | 극소수 VIP | 1:1 컨설팅, 한정판, 프라이빗 이벤트 |

**설계 규칙:**
- 각 단계 사이에 자연스러운 업셀/크로스셀 경로 명시
- 각 단계의 "고객이 얻는 가치"를 구체적으로 정의
- 현재 브랜드의 가치 사다리 갭(빠진 단계) 식별
- ⛔ 안티패턴: 단순 가격 티어 나열 → 가치 상승 논리가 반드시 포함

### 4-B. 매력적 캐릭터 (Attractive Character) 정의

**출처: DotCom Secrets + Expert Secrets**

브랜드가 고객과 소통할 때의 페르소나를 설계한다.

**4가지 캐릭터 유형 중 선택:**
| 유형 | 특징 | 적합한 브랜드 |
|------|------|-------------|
| 리더 (Leader) | 앞서 걸어가며 따라오라고 하는 사람 | 전문가/권위 기반 브랜드 |
| 모험가 (Adventurer) | 함께 탐험하자는 사람 | 혁신/탐험 기반 브랜드 |
| 전달자 (Reporter) | 조사해서 알려주는 사람 | 교육/정보 기반 브랜드 |
| 영웅/반항아 (Reluctant Hero) | 자기도 원치 않았지만 해야 했던 사람 | 스토리/공감 기반 브랜드 |

**정의 항목:**
1. 캐릭터 유형 선택 + 근거
2. 백스토리 (Origin Story) — 왜 이 일을 시작했는가
3. 페르소나 특성 — 말투, 가치관, 적(Villain) 정의
4. 에피파니 브릿지 (Epiphany Bridge) — 고객이 "아하!" 할 전환 스토리
5. 결점 (Flaws) — 완벽하지 않은 점 (공감 요소)

### 4-C. 후크-스토리-오퍼 (Hook-Story-Offer) 구조

**출처: Expert Secrets + Traffic Secrets**

모든 마케팅 커뮤니케이션의 기본 3요소를 설계한다.

| 요소 | 역할 | 설계 기준 |
|------|------|----------|
| 후크 (Hook) | 스크롤/이탈을 멈추게 하는 첫 한마디 | 3초 내 주의 포착. 호기심/공포/이익/충격 중 택1 |
| 스토리 (Story) | 가치를 인지시키고 욕구를 키우는 서사 | 매력적 캐릭터의 에피파니 브릿지 활용. 감정→논리 순서 |
| 오퍼 (Offer) | 구매/행동을 유도하는 제안 | 가치 사다리의 해당 단계 오퍼. 긴급성+희소성+보너스 |

**채널별 HSO 설계:**
- SNS 광고용 (짧은 버전)
- 랜딩페이지용 (긴 버전)
- 이메일 시퀀스용 (연속 버전)
- 콘텐츠 마케팅용 (교육 버전)

각 채널별 후크 3개 이상, 스토리 프레임 1개, 오퍼 구성 1개를 구체적으로 작성.

### 4-D. 드림100 (Dream 100) 리스트

**출처: Traffic Secrets**

타겟 고객이 이미 모여 있는 100개 접점을 식별하고, 침투 전략을 설계한다.

**카테고리별 드림100 분류:**

| 카테고리 | 예시 | 탐색 방법 |
|---------|------|----------|
| 인플루언서 | 뷰티 유튜버, 인스타 셀럽 | WebSearch: "{카테고리} 인플루언서 TOP" |
| 미디어/채널 | 유튜브 채널, 팟캐스트, 뉴스레터 | WebSearch: "{카테고리} 미디어 채널" |
| 커뮤니티 | 네이버 카페, 레딧, 디시인사이드 | WebSearch: "{카테고리} 커뮤니티" |
| 플랫폼 | 올리브영, 쿠팡, 무신사 | 카테고리별 유통 채널 |
| 이벤트/컨퍼런스 | 코엑스 전시, 세미나 | WebSearch: "{카테고리} 전시 컨퍼런스" |
| 제휴/파트너 | 이종 브랜드, 보완재 | 가치 사다리 연계 |

**드림100 실행 전략:**
각 드림100 접점에 대해:
1. 접점명 + URL/채널
2. 추정 도달 규모 (팔로워/구독자/회원 수)
3. 침투 방식: Earn(무료 노출) / Buy(광고) / Partner(협업)
4. 우선순위: ⭐1~3 (도달 규모 × 타겟 적합도 × 진입 난이도)
5. 예상 비용 범위

**드림100은 100개를 채우지 않아도 됨.** 카테고리별 핵심 5~10개씩, 총 30~50개를 현실적으로 선정.

## Step 5: 통합 전략 대시보드 제작

### /viz 연동 규칙

/strategy는 /viz 스킬을 직접 호출하지 않는다.
/viz의 HTML 스켈레톤 구조, CCFM 브랜드 변수, Chart.js 패턴만 차용한다.
탭 설계, 프레임워크 시각화, 데이터 바인딩은 /strategy가 직접 담당.

/viz 스킬의 스켈레톤 템플릿과 CCFM 브랜드 규칙을 적용하여 대시보드를 제작합니다.

### 대시보드 구조

```
[헤더] 제목 + 날짜 + 브랜드명 + 전략 목적

[탭 네비게이션]
  ├── 탭1: 전략 요약 (Executive Summary)
  ├── 탭2: 매슬로 욕구 분석
  ├── 탭3: 고객 창조 스코어카드
  ├── 탭4: 가치 사다리 & 오퍼 설계
  ├── 탭5: 매력적 캐릭터 & HSO
  ├── 탭6: 드림100 맵
  └── 탭7: 실행 로드맵

[필터 바] 시간축 필터 (즉시/단기/중장기) + 프레임워크 필터
```

### 필수 섹션

#### 탭1: Executive Summary
- KPI 카드 그리드:
  - 매슬로 주력 욕구 단계 + 키워드
  - 드러커 고객 창조 점수 (X/5)
  - 가치 사다리 갭 수 (빠진 단계)
  - 드림100 선정 수 + TOP 3 채널
- 한줄 전략 방향 (큰 글씨 하이라이트)
- 핵심 인사이트 3가지 카드

#### 탭2: 매슬로 욕구 분석
- 5단계 피라미드 시각화 (CSS 기반, 현재 강점/기회 영역 색상 구분)
- 경쟁사 대비 욕구 포지션 레이더 차트 (Chart.js radar)
- 욕구 단계별 메시지 전략 카드
- 욕구 갭 분석 바 차트

#### 탭3: 고객 창조 스코어카드
- 5대 진단 항목 게이지 차트 / 프로그레스 바
- 항목별 상세 카드 (현재 상태 + 근거 + 개선 방향)
- 종합 점수 도넛 차트

#### 탭4: 가치 사다리 & 오퍼
- 가치 사다리 시각화 (계단형 CSS, 각 단계 클릭 시 상세 확장)
- 단계별 오퍼 상세 카드 (가격, 가치, 업셀 경로)
- 현재 vs 설계 비교 (Before/After)
- LTV 시뮬레이션 (가치 사다리 완성 시 예상 LTV 증가)

#### 탭5: 매력적 캐릭터 & HSO
- 캐릭터 프로필 카드 (유형, 백스토리, 페르소나)
- 에피파니 브릿지 타임라인 (CSS 타임라인)
- 채널별 HSO 매트릭스 (후크/스토리/오퍼를 채널별로 정리)
- 후크 예시 갤러리 (카드형, 클릭 시 전체 HSO 확장)

#### 탭6: 드림100 맵
- 카테고리별 드림100 리스트 (접이식 카드)
- 우선순위 버블 차트 (X축: 도달 규모, Y축: 타겟 적합도, 버블 크기: 진입 난이도 역수)
- 침투 방식별 필터 (Earn / Buy / Partner)
- 예상 비용 스택 바 차트

#### 탭7: 실행 로드맵
- 타임라인 시각화 (즉시 / 단기 / 중장기)
- 각 액션의 필수 필드:

| 필드 | 필수 | 설명 |
|------|------|------|
| 소스 프레임 | ✅ | 어떤 프레임워크에서 도출 (예: 매슬로 L3×드림100, 드러커 Q2) |
| 채널 | ✅ | 실행 채널 |
| 예산 범위 | ✅ | 월 단위 예상 예산 |
| 타임라인 | ✅ | 시작~완료 기간 |
| KPI | ✅ | 측정 지표 + 목표치 |
| 가치 사다리 단계 | ✅ | 어느 단계의 오퍼인지 |

- **즉시 액션 (1~2주)**: 기존 리소스로 바로 실행 가능한 3가지
- **단기 액션 (1~3개월)**: 예산/리소스 확보 후 실행

**⛔ 액션 안티패턴 (추상적 표현 금지):**
- "브랜드 인지도 제고", "고객 경험 개선", "SNS 활성화"
