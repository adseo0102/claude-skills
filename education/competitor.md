# CCFM 경쟁사 분석 리포트

사용자가 $ARGUMENTS 로 분석 대상과 기준을 전달합니다.
예시: `/competitor 무신사 vs 29CM 비교 분석. 타겟 고객, 가격 전략 중심으로.`

아래 파이프라인을 **반드시 순서대로** 실행하세요.

---

## Step 1: 요구사항 분석

사용자 입력에서 아래를 추출합니다:

| 항목 | 추출 |
|------|------|
| 자사 | 우리 회사/브랜드 (있으면) |
| 경쟁사 | 비교 대상 1~5개 |
| 비교 기준 | 가격, 타겟, 채널, 기능, 포지셔닝 등 |
| 산업/카테고리 | 이커머스, 뷰티, SaaS 등 |
| 출력 형태 | 비교표, SWOT, 대시보드 등 |

---

## Step 2: 리서치

### 2-1. WebSearch

각 경쟁사에 대해 최소 3회 검색:
- "{경쟁사} 서비스 소개" / "{경쟁사} 가격 모델"
- "{경쟁사} vs {경쟁사} 비교"
- "{산업} 시장 점유율 2025"

### 2-2. Gemini 교차검증 (gemini-tools 있을 경우)

```bash
node ~/gemini-tools/gemini-search.mjs "{경쟁사} {산업} 시장 현황"
```

### 리서치 규칙
- 핵심 수치는 2개 이상 출처 교차 검증
- 1년 이내 데이터 우선
- 출처 반드시 명시
- Gemini 실패 시 WebSearch만으로 진행

수집 완료 후 아래 형식으로 사용자에게 보여줍니다:

```
## 리서치 결과: {비교 주제}

| 항목 | {자사} | {경쟁사A} | {경쟁사B} |
|------|--------|-----------|-----------|
| 주요 서비스 | | | |
| 타겟 고객 | | | |
| 가격 모델 | | | |
| ... | | | |

### 핵심 인사이트
### 분석 앵글 제안
```

## Step 4: 분석 리포트 + 시각화 제작

/viz 스킬의 CCFM 브랜드 스타일을 적용하여 인터랙티브 대시보드를 만듭니다.

### 필수 포함 섹션

1. **헤더**: 리포트 제목 + 분석 일자 + 대상 기업
2. **비교 매트릭스**: 항목별 비교표 (테이블, 필터 가능)
3. **SWOT 분석**: 각 기업별 강점/약점/기회/위협 (2x2 그리드)
4. **시각화 차트** (Chart.js):
   - 레이더 차트: 역량 비교 (5~7 항목)
   - 바 차트: 정량적 비교 (가격, 규모, 점유율)
   - 도넛 차트: 시장 점유율 (해당 시)
5. **포지셔닝 맵**: 2축 기반 위치 (예: 가격 vs 품질)
6. **핵심 인사이트**: 3~5개 불릿, 액션 제안 포함
7. **출처**: 리서치 출처 목록

### 디자인 규칙
- /viz 스킬의 스켈레톤 템플릿 기반
- CCFM 브랜드 컬러 (#e5007e 핑크 accent)
- Pretendard 폰트
- Chart.js: `Chart.defaults.animation = false`, 테마 토글 시 차트 재생성
- 다크/라이트 테마 토글
- 반응형 (모바일 대응)
- 메뉴 (PNG 다운로드, 인쇄)
- 스크롤 리빌 애니메이션

### 파일 저장
- `~/Downloads/{분석주제-케밥케이스}.html`에 저장
- `start` 명령으로 브라우저 열기

---

## Step 5: Vercel 배포 (자동)

1. **프로젝트명**: 한글 의미를 영어로 번역
   - 접두사: `ccfm-competitor-`
   - 예: "무신사 vs 29CM" → `ccfm-competitor-musinsa-29cm`
2. **배포**:
   ```
   mkdir -p $TEMP/{프로젝트명}
   cp ~/Downloads/{파일명}.html $TEMP/{프로젝트명}/index.html
   cd $TEMP/{프로젝트명}
   vercel --yes --prod
   ```
3. **URL 제공**: `✅ 배포 완료: https://{프로젝트명}.vercel.app`
